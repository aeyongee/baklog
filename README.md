# Baklog

일잘러로 거듭나기 위한 작업 관리 서비스입니다 ⏰

할 일을 입력하면 AI가 아이젠하워 매트릭스 기반으로 자동 분류합니다.
사용자는 분류 결과를 확인하고, 필요하면 조정한 뒤 오늘 하루를 실행하면 됩니다.

## 핵심 컨셉

- **입력은 빠르게** — 한 줄씩 할 일만 적으면 끝
- **분류는 AI가** — 중요도/긴급도를 사용자 대신 판단
- **결정은 하루에 한 번** — 매일 아침 한 번만 확인하고 실행
- **놓치는 일 없이** — 미완료 작업은 자동 이월, 장기 작업은 백로그에서 관리

## 주요 기능

### 온보딩 설문

최초 가입 시 간단한 설문(업무 스타일, 중요도 기준 등)을 진행합니다.
답변을 바탕으로 AI 분류가 사용자에게 맞춰집니다.

### 할 일 입력

할 일을 한 줄씩 입력하고, 준비되면 "AI 분류하기"를 누릅니다.
입력한 작업은 분류 전에 자유롭게 삭제할 수 있습니다.

### AI 자동 분류

입력된 작업들을 AI가 한 번에 분석하여 4개 사분면으로 분류합니다.

| 사분면 | 의미                                         |
| ------ | -------------------------------------------- |
| Q1     | 긴급하고 중요한 일 — 지금 당장               |
| Q2     | 중요하지만 급하지 않은 일 — 계획적으로       |
| Q3     | 급하지만 크게 중요하지 않은 일 — 빠르게 처리 |
| Q4     | 급하지도 중요하지도 않은 일 — 여유 있을 때   |

각 작업에 대해 AI의 판단 이유와 확신도를 함께 보여줍니다.

### 분류 결과 검토

AI 분류 결과를 확인하고, 동의하지 않으면 중요/긴급 토글로 직접 조정합니다.
확정하면 오늘의 대시보드가 생성됩니다.

### 오늘의 대시보드

두 가지 뷰로 오늘 할 일을 관리합니다.

- **리스트 뷰** — 작업을 순서대로 완료 체크
- **사분면 뷰** — 아이젠하워 매트릭스로 한눈에 파악, 드래그 앤 드롭으로 사분면 간 이동 가능

완료된 작업은 접었다 펼 수 있고, 불필요한 작업은 폐기할 수 있습니다.

### 자동 이월

오늘 끝내지 못한 작업은 다음 날 자동으로 넘어옵니다.
다음 날 진입 시 이월 작업 확인 팝업이 표시되며, 각 작업을 오늘 계획에 포함하거나 백로그로 보낼 수 있습니다.

### 백로그

장기간 미완료된 작업은 룰 엔진에 의해 백로그로 자동 이동됩니다.
백로그에서 다시 오늘의 작업으로 가져올 수 있고, 오늘 페이지에서도 백로그 알림을 통해 확인할 수 있습니다.

### 월간 뷰

캘린더 형태로 날짜별 작업 완료 현황을 확인할 수 있습니다.
한 달간의 작업 흐름을 한눈에 파악할 수 있습니다.

### 다크모드

시스템 설정에 연동되는 다크모드를 지원합니다.
수동으로 라이트/다크 모드를 전환할 수도 있습니다.

## 기술 스택

| 영역         | 기술                             |
| ------------ | -------------------------------- |
| 프레임워크   | Next.js (App Router), TypeScript |
| 스타일링     | Tailwind CSS                     |
| 데이터베이스 | PostgreSQL, Prisma ORM           |
| 인증         | Auth.js (Google 소셜 로그인)     |
| AI           | OpenAI ChatGPT API               |
| 배포         | Vercel                           |
| 분석         | Vercel Analytics                 |

## 로컬 개발

```bash
# 패키지 설치
npm install

# 환경변수 설정
cp .env.example .env
```

`.env` 파일에 아래 값을 설정합니다.

| 변수                 | 설명                                        |
| -------------------- | ------------------------------------------- |
| `DATABASE_URL`       | PostgreSQL 연결 문자열                      |
| `AUTH_SECRET`        | Auth.js 시크릿 (`npx auth secret`으로 생성) |
| `AUTH_GOOGLE_ID`     | Google OAuth Client ID                      |
| `AUTH_GOOGLE_SECRET` | Google OAuth Client Secret                  |
| `OPENAI_API_KEY`     | OpenAI API Key                              |

```bash
# DB 마이그레이션
npx prisma migrate dev

# 개발 서버 실행
npm run dev
```

http://localhost:3000 에서 확인할 수 있습니다.

## 배포

Vercel에 연결하면 `main` 브랜치 푸시 시 자동 배포됩니다.
빌드 과정에서 Prisma Client 생성과 DB 마이그레이션이 자동으로 실행됩니다.
